# Package Management: Cargo vs NuGet

> **What you'll learn:** `Cargo.toml` vs `.csproj`, version specifiers, `Cargo.lock`, feature flags for conditional compilation, and common Cargo commands.
>
> **Difficulty:** Beginner

### Dependency Declaration
In C#, you use `.csproj` or `NuGet.config`. In Rust, you use `Cargo.toml`.

#### C# .csproj
```xml
<PackageReference Include="Newtonsoft.Json" Version="13.0.3" />
```

#### Rust Cargo.toml
```toml
[dependencies]
serde_json = "1.0"
```

---

## Versioning and the Lock File
Rust uses Semantic Versioning (SemVer) by default.
*   **`"1.0"`**: Means `>=1.0.0` and `<2.0.0`.
*   **`Cargo.lock`**: This file is automatically generated and ensures that every person working on the project uses the **exact same version** of every dependency. This is similar to `packages.lock.json` in modern .NET project.

---

## Feature Flags: Conditional Compilation
One of Cargo's most powerful features is **Features**. It allows you to make dependencies or code blocks optional, reducing compile times and binary size.

```toml
# Cargo.toml
[dependencies]
serde = { version = "1.0", features = ["derive"] }
```

In your code, you can use `#[cfg(feature = "...")]` to conditionally include code. This is a much cleaner alternative to C#'s `#if DEBUG` preprocessor directives.

## Common Commands Comparison

| **Action** | **`dotnet` Command** | **`cargo` Command** |
| :--- | :--- | :--- |
| **Build Project** | `dotnet build` | `cargo build` |
| **Run Project** | `dotnet run` | `cargo run` |
| **Test Project** | `dotnet test` | `cargo test` |
| **Add Package** | `dotnet add package X` | `cargo add X` |
| **Update Packages** | `dotnet restore` | `cargo update` |

---

## Popular Crate Alternatives
| **C# Library** | **Rust Equivalent** | **Purpose** |
| :--- | :--- | :--- |
| **Newtonsoft.Json** | `serde_json` | Serialization |
| **HttpClient** | `reqwest` | HTTP Requests |
| **Entity Framework** | `diesel` or `sqlx` | Database/ORM |
| **Serilog / NLog** | `tracing` or `log` | Logging |
| **xUnit / NUnit** | Built-in `#[test]` | Testing |

## Exercise: Add a Dependency
**Challenge:** Add the `chrono` library for date/time handling to your project and enable the `serde` feature.

**Solution:**
```toml
[dependencies]
chrono = { version = "0.4", features = ["serde"] }
```
**Takeaway:** Cargo is more than just a package manager; it's a build system and test runner that provides a unified, delightful experience for Rust developers.
