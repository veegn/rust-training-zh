# 9. Smart Pointers and Interior Mutability 🟡

> **What you'll learn:**
> - `Box`, `Rc`, `Arc` for heap allocation and shared ownership.
> - `Weak` references for breaking reference cycles.
> - `Cell`, `RefCell`, and `Cow` for interior mutability.
> - `Pin` for self-referential types.

## Box, Rc, Arc — Heap Allocation and Sharing

| Pointer | Use Case | Mutability | Thread-Safe |
|---------|----------|------------|-------------|
| `Box<T>` | Single owner on heap. | Via `&mut` | ✅ (if T: Send) |
| `Rc<T>` | Shared ownership (single thread). | None (wrap in RefCell) | ❌ |
| `Arc<T>` | Shared ownership (multi-thread). | None (wrap in Mutex) | ✅ |

### Weak References

`Weak<T>` is a non-owning handle. Use it to break cycles (e.g., child → parent pointers) and prevent memory leaks.

```rust
let parent = Rc::new(Node { ... });
let child = Rc::new(Node {
    parent: Rc::downgrade(&parent), // Weak link
});
```

## Interior Mutability

Interior mutability allows you to mutate data behind a shared (`&`) reference by moving the borrow check to runtime.

- **`Cell<T>`**: No panics, works by replacing/copying values. Only for `Copy` types.
- **`RefCell<T>`**: Works for any type, but panics if you violate borrow rules at runtime.

```rust
let data = RefCell::new(vec![1, 2, 3]);
data.borrow_mut().push(4); // Works on &RefCell
```

## Cow — Clone on Write

`Cow` (Clone on Write) allows you to avoid allocating a new `String` or `Vec` if you don't actually need to modify the data.

```rust
fn normalize(s: &str) -> Cow<'_, str> {
    if s.contains('\t') {
        Cow::Owned(s.replace('\t', " "))
    } else {
        Cow::Borrowed(s)
    }
}
```

## Pin — Preventing Moves

`Pin<P>` ensures that the value pointed to will never be moved in memory. This is required for **self-referential types** (like those generated by `async/await`).

- **`Unpin`**: Most types are `Unpin` (safe to move).
- **`!Unpin`**: Types that must stay at one memory address once initialized.

---

## Drop Ordering (RFC 1857)

- **Local variables**: Dropped in **reverse** declaration order.
- **Struct fields**: Dropped in **forward** declaration order (top to bottom).
